KDataPy Samba Utility Functions¶

KDataPy.samba_utilities.datetimeobjectfromsamba(date)[source]¶

returns a datetime object for a given Samba date format year-month-day string. For example, the input argument, date, must look something like ma22

KDataPy.samba_utilities.getcurrentsambayear()[source]¶

returns the Samba date format year string value for right now.

KDataPy.samba_utilities.isvalidsambadatafilename(name)[source]¶

Test if name is a valid name for Samba file (aka partition) names. Such as ma22000_012

KDataPy.samba_utilities.isvalidsambadate(sambadate)[source]¶

Test if sambadate is a valid date for Samba file names. Such as ma22, or lk14.

KDataPy.samba_utilities.isvalidsambametadatafilename(name)[source]¶

Test if name is a valid name for Samba meta data file names. Such as ma22000_BB (or ntp or log)

KDataPy.samba_utilities.isvalidsambarunname(name)[source]¶

Test if name is a valid name for Samba run names. Such as ma22000

KDataPy.samba_utilities.monthfromsamba(month)[source]¶

returns month number between 1 and 12 for a give Samba date format month string. The input argument, month, must be a string between a and l, inclusively.

KDataPy.samba_utilities.sambadatafileregex()[source]¶

returns a regular expression string that matches a samba file (aka ‘partition’) string. something like ‘ma22a012_023’, for example. you can use this in the python re module.

KDataPy.samba_utilities.sambadate(year, month, day)[source]¶

returns the Samba date format year-month0day string value for a given year, month and day. For example, if year = 2012, and month = 2, and day = 12 this function will return ‘mb12’. The input arguments follow the rules of getsambayear, and getsambamonth.

KDataPy.samba_utilities.sambadatenow()[source]¶

returns the Samba date format year-month-day string value for right now.

KDataPy.samba_utilities.sambadateregex()[source]¶

returns a regular expression string that matches a samba date string. something like ‘ma22’, for example. you can use this in the python re module.

KDataPy.samba_utilities.sambametadatafileregex()[source]¶

returns a regular expression string that matches a samba meta data file string. something like ‘ma22a012_BB’, ‘_ntp’, or ‘_log’ for example. It only matches _BB, _ntp and _log. you can use this in the python re module.

KDataPy.samba_utilities.sambamonth(month)[source]¶

returns the Samba date format month string value for a given month. For example, if month = 1, this function will return ‘a’. The input argument, month, must be number from 1 to 12

KDataPy.samba_utilities.sambamonthnow()[source]¶

returns the Samba date format month string value for right now.

KDataPy.samba_utilities.sambarunnameregex()[source]¶

returns a regular expression string that matches a samba run string. something like ‘ma22a012’, for example. you can use this in the python re module.

KDataPy.samba_utilities.sambayear(year)[source]¶

returns the Samba date format year string value for a given year. For example, if year = 2012, this function will return ‘m’ The input arguemnt, year, must be greater than or equal to 2000 and less than 2026

KDataPy.samba_utilities.sambayearmonth(year, month)[source]¶

returns the Samba date format year-month string value for a given year and month. For example, if year = 2012, and month = 2, this function will return ‘mb’. The input arguments follow the rules of getsambayear, and getsambamonth.

KDataPy.samba_utilities.sambayearmonthnow()[source]¶

returns the Samba date format year-month string value for right now.

KDataPy.samba_utilities.yearfromsamba(year)[source]¶

returns year number between 2000 and 2025 for a give Samba date format year string. The input argument, year, must be a string between a and z, inclusively.
